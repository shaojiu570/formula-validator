# 📖 公式验证器使用指南

## 🎯 快速开始

### 方式一：本地开发（推荐用于开发和测试）

```bash
# 1. 安装依赖（首次使用）
npm install

# 2. 启动开发服务器
npm run dev

# 3. 浏览器自动打开 http://localhost:3000
```

### 方式二：构建后使用（推荐用于个人使用）

```bash
# 1. 构建项目
npm run build

# 2. 预览构建结果
npm run preview

# 3. 访问 http://localhost:4173
```

### 方式三：直接打开（最简单）

```bash
# 1. 构建项目
npm run build

# 2. 直接用浏览器打开
# 打开 dist/index.html 文件即可
```

## 📁 文件说明

### 核心文件
- `index.html` - 入口页面
- `vite.config.js` - 构建配置
- `package.json` - 项目依赖

### 源码目录（src/）
- `main.js` - 应用入口
- `App.vue` - 根组件
- `components/` - 可复用组件
- `views/` - 页面组件
- `stores/` - 状态管理（公式逻辑）
- `router/` - 路由配置
- `styles/` - 全局样式

### 构建输出（dist/）
- 构建后的生产文件
- 可以直接部署或打开使用

## 🔧 常见操作

### 修改公式元素

编辑 `src/stores/formula.js`：

```javascript
const elements = [
  '01', '02', '03', // ... 修改这里
]
```

### 调整界面样式

编辑 `src/styles/global.css`：

```css
/* 修改主题颜色、字体大小等 */
```

### 修改验证逻辑

编辑 `src/stores/formula.js` 中的 `validateFormula` 方法。

## 💾 数据存储

所有数据（收藏、历史、设置）都保存在浏览器的 localStorage 中：
- 无需数据库
- 无需后端服务器
- 数据仅保存在本地浏览器

## 🌐 部署方式

### 1. 本地使用（最简单）
直接打开 `dist/index.html` 文件

### 2. 局域网共享
```bash
npm run preview -- --host
# 其他设备通过 IP 地址访问
```

### 3. 静态服务器
将 `dist/` 目录上传到任何静态服务器：
- GitHub Pages
- Vercel
- Netlify
- 自己的服务器

## ❓ 常见问题

### Q: 为什么要运行 npm install？
A: 安装项目依赖（Vue、Element Plus 等）

### Q: 可以不用命令行吗？
A: 可以！构建后直接打开 `dist/index.html`

### Q: 数据会丢失吗？
A: 数据保存在浏览器本地，除非清除浏览器数据

### Q: 可以离线使用吗？
A: 可以！构建后的文件完全离线可用

### Q: 如何备份数据？
A: 浏览器开发者工具 → Application → Local Storage → 导出

## 🎨 界面说明

### 主要功能区
1. **公式输入** - 输入要验证的公式
2. **参数设置** - 设置验证期数、命中期数等
3. **验证结果** - 四层统计显示
4. **公式库** - 预设公式模板
5. **搜索生成** - 智能生成公式组合
6. **收藏夹** - 保存常用公式
7. **历史记录** - 查看验证历史

### 统计显示层级
1. **第一层** - 公式列表及命中模式
2. **第二层** - 近期开出次数统计
3. **第三层** - 结果类型分组
4. **第四层** - 码类结果汇总

---

**提示**: 这是一个纯前端应用，无需任何后端服务，开箱即用！
